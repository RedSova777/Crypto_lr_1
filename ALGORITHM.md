# Быстрый алгоритм поиска дроби слева от 3/7

## 1. Постановка задачи

Требуется среди всех несократимых правильных дробей `n/d`, для которых выполняется:

```text
0 < n < d
НОД(n, d) = 1
d <= 1 000 000
```

найти наибольшую дробь, которая строго меньше `3/7`.

Иными словами, необходимо найти непосредственного левого соседа дроби `3/7` в последовательности Фарея порядка `N = 1 000 000`.

---

## 2. Ключевое свойство соседних дробей Фарея

Для двух соседних дробей

```text
n/d < a/b
```

в последовательности Фарея выполняется равенство:

```text
a*d - b*n = 1
```

Для целевой дроби `a/b = 3/7` получаем:

```text
3*d - 7*n = 1
```

Отсюда:

```text
7*n = 3*d - 1
n = (3*d - 1) / 7
```

Чтобы `n` было целым числом, выражение `3*d - 1` должно делиться на 7 без остатка. Это можно записать так:

```text
3*d ≡ 1 (mod 7)
```

Обратный элемент к 3 по модулю 7 равен 5, поскольку:

```text
3*5 = 15
15 ≡ 1 (mod 7)
```

Следовательно:

```text
d ≡ 5 (mod 7)
```

То есть знаменатель должен давать остаток 5 при делении на 7.

---

## 3. Общий вид подходящих знаменателей

Все числа, дающие остаток 5 при делении на 7, имеют вид:

```text
d = 5 + 7*k
```

где `k` — целое неотрицательное число.

Первые такие знаменатели:

```text
5, 12, 19, 26, 33, ...
```

Для каждого такого знаменателя числитель вычисляется по формуле:

```text
n = (3*d - 1) / 7
```

Например, при `d = 5`:

```text
n = (3*5 - 1) / 7
n = 14 / 7
n = 2
```

Получаем дробь `2/5`, которая действительно стоит слева от `3/7` в последовательности Фарея порядка 8.

---

## 4. Выбор максимального знаменателя

Чтобы дробь была как можно ближе к `3/7`, необходимо взять максимально возможный знаменатель вида:

```text
d = 5 + 7*k
```

но при этом должно выполняться ограничение:

```text
d <= N
```

Подставляем общий вид знаменателя:

```text
5 + 7*k <= N
```

Отсюда:

```text
7*k <= N - 5
k <= (N - 5) / 7
```

Так как `k` должно быть целым, берём целую часть результата:

```text
k = floor((N - 5) / 7)
```

При `N = 1 000 000`:

```text
k = floor((1 000 000 - 5) / 7)
k = floor(999 995 / 7)
k = 142 856
```

Теперь вычисляем знаменатель:

```text
d = 5 + 7*k
d = 5 + 7*142 856
d = 999 997
```

Следующее подходящее значение было бы равно:

```text
999 997 + 7 = 1 000 004
```

Оно уже превышает ограничение `1 000 000`.

---

## 5. Вычисление числителя

Используем формулу:

```text
n = (3*d - 1) / 7
```

Подставляем `d = 999 997`:

```text
n = (3*999 997 - 1) / 7
n = (2 999 991 - 1) / 7
n = 2 999 990 / 7
n = 428 570
```

Искомая дробь:

```text
428570/999997
```

Искомый числитель:

```text
428570
```

---

## 6. Дробь находится слева от 3/7

Для найденных чисел выполняется:

```text
3*d - 7*n = 1
```

Подставим значения:

```text
3*999 997 - 7*428 570 = 1
```

Следовательно:

```text
3*d > 7*n
```

После деления обеих частей на положительное число `7*d` получаем:

```text
n/d < 3/7
```

Значит, найденная дробь действительно расположена слева от `3/7`.

---

## 7. Дробь несократима

Дробь называется несократимой, если её числитель и знаменатель не имеют
общего делителя, кроме 1:

```text
НОД(n, d) = 1
```

Для найденных чисел выполняется равенство:

```text
3*d - 7*n = 1
```

В нашем случае:

```text
3*999 997 - 7*428 570 = 1
```

Разберём, почему из этого равенства следует несократимость дроби.

### 7.1. Предположим обратное

Предположим, что числитель `n` и знаменатель `d` всё-таки имеют общий
делитель `g > 1`.

Это означает, что существуют целые числа `n1` и `d1`, для которых:

```text
n = g*n1
d = g*d1
```

То есть и числитель, и знаменатель содержат общий множитель `g`.

### 7.2. Подставим это в равенство

Подставляем:

```text
n = g*n1
d = g*d1
```

в выражение:

```text
3*d - 7*n
```

Получаем:

```text
3*(g*d1) - 7*(g*n1)
```

Выносим общий множитель `g` за скобки:

```text
g*(3*d1 - 7*n1)
```

Следовательно, если `g` делит и `n`, и `d`, то всё выражение
`3*d - 7*n` также обязательно делится на `g`.

Это частный случай общего свойства:

> Если число `g` делит два целых числа, то оно делит любую их целочисленную
> линейную комбинацию.

В данном случае линейная комбинация имеет вид:

```text
3*d + (-7)*n
```

### 7.3. Возникает противоречие

С одной стороны, мы получили, что:

```text
g делит (3*d - 7*n)
```

С другой стороны:

```text
3*d - 7*n = 1
```

Значит, число `g` должно делить 1:

```text
g делит 1
```

Но у числа 1 есть только два целых делителя:

```text
1 и -1
```

Положительный общий делитель может быть только равен 1.

Это противоречит нашему предположению:

```text
g > 1
```

Следовательно, общего делителя больше 1 у `n` и `d` нет.

Поэтому:

```text
НОД(n, d) = 1
```

и дробь:

```text
428570/999997
```

является несократимой.

---

## 8. Между найденной дробью и 3/7 нет другой допустимой дроби

Для соседних дробей Фарея существует критерий:

```text
a*d - b*n = 1
```

и одновременно:

```text
b + d > N
```

В нашем случае:

```text
a = 3
b = 7
n = 428 570
d = 999 997
N = 1 000 000
```

Проверяем первое условие:

```text
3*999 997 - 7*428 570 = 1
```

Проверяем второе условие:

```text
7 + 999 997 = 1 000 004
1 000 004 > 1 000 000
```

Оба условия выполнены.

Следовательно, дроби `428570/999997` и `3/7` являются соседними в последовательности Фарея порядка `1 000 000`. Это означает, что между ними нет другой несократимой дроби со знаменателем не более `1 000 000`.

## 9. Общий алгоритм для произвольной дроби a/b

Пусть задана несократимая правильная дробь `a/b` и максимальный знаменатель `N`, причём:

```text
0 < a < b
НОД(a, b) = 1
N >= b
```

Для левого соседа `n/d` требуется решить уравнение:

```text
a*d - b*n = 1
```

### Шаг 1. Найти начальный знаменатель d0

Из уравнения следует:

```text
a*d ≡ 1 (mod b)
```

Поэтому `d0` — это обратный элемент к `a` по модулю `b`:

```text
d0 = a^(-1) mod b
```

### Шаг 2. Найти начальный числитель n0

```text
n0 = (a*d0 - 1) / b
```

### Шаг 3. Записать все решения

Все целочисленные решения уравнения `a*d - b*n = 1` имеют вид:

```text
d = d0 + k*b
n = n0 + k*a
```

где `k` — целое неотрицательное число.

### Шаг 4. Выбрать максимальное k

Из ограничения `d <= N` следует:

```text
d0 + k*b <= N
k <= (N - d0) / b
```

Поэтому:

```text
k = floor((N - d0) / b)
```

### Шаг 5. Вычислить итоговую дробь

```text
n = n0 + k*a
d = d0 + k*b
```

## 10. Применение общего алгоритма к дроби 3/7

Исходные данные:

```text
a = 3
b = 7
N = 1 000 000
```

Обратный элемент к 3 по модулю 7:

```text
d0 = 5
```

Начальный числитель:

```text
n0 = (3*5 - 1) / 7
n0 = 2
```

Все решения:

```text
d = 5 + 7*k
n = 2 + 3*k
```

Максимальное значение `k`:

```text
k = floor((1 000 000 - 5) / 7)
k = 142 856
```

Итоговые значения:

```text
d = 5 + 7*142 856 = 999 997
n = 2 + 3*142 856 = 428 570
```

Ответ:

```text
428570
```

---

## 11. Сложность алгоритма

Для нахождения обратного элемента используется расширенный алгоритм Евклида.

Его временная сложность:

```text
O(log b)
```

После нахождения обратного элемента выполняется постоянное количество арифметических операций.

Итоговая сложность:

```text
Время:  O(log b)
Память: O(1)
```

Для конкретной дроби `3/7` число операций фактически постоянно и не зависит от `N`.
